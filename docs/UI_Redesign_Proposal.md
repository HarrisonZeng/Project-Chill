# UI Redesign Proposal — Project Chill

Stage 1 设计提案（无代码）。Stage 2 在用户确认本文后再开始。

基准分辨率：1600 × 900（`stretch/mode = canvas_items`）。
所有比例与坐标以此分辨率为参考；缩放由 Godot 的 stretch 系统处理。

---

## 1. 框架概念

**Yua + 她身后的房间继续占满 100% 屏幕，不裁切、不加外框。** 这一版重做不是"把 Yua 装进一个 call frame"，而是**把现有 UI 全部重做成漂浮在通话画面之上的 cozy frosted overlay**——参考全屏 FaceTime / Zoom 的"对方填满整个屏幕、控件只是漂浮 UI"模型。

通话感靠**三类轻量元素**点出来，而不是靠任何裁切或外框：

- **左上角的通话状态 pill**：一个绿色呼吸点 + `Yua · 已连接 · 00:28:14`，把当前那堆"日期 + 时间 + Night"大字消化成一行 12-14px 的单行。**这是"我加入了一场通话"的最小可读信号。**
- **底部居中的 call control bar**：6 个 ghost icon——`🎤 静音 / 🔈 音量 / 🖥 共享屏幕 / 💬 聊天 / ⚙ 设置 / 📞 结束通话`。默认 40% 透明度漂浮，hover 才完全显现；对话激活时进一步淡出。这是这次新加的"通话感"主要载体，**装饰性 > 功能性**，但功能可以慢慢接上（mute → Yua 语音静音、screenshare → 后续屏幕共享玩法钩子、chat → 折叠/展开对话区、settings → 现在的设置面板、end call → 退出确认、volume → BGM 音量）。
- **右下角的"参与者缩略 chip"**（**可选，默认关闭**）：一个 `你 · 静音中` 的小圆角条，强化"两个人在通话"的感觉。**不放头像、不放摄像头**——只一行文字 + 麦克风 icon。Mockup 4 会画出来给你选择是否启用。

**所有原有的生产力 widget**（focus timer / tasks / music / dialogue / settings）**保持当前的大致位置基本盘**，**只做样式重做 + 主次折叠**，不大改布局逻辑。所以这次改动的风险被压在"样式 + 折叠交互"层，不动业务逻辑。

**单一展开规则**：任何"展开态"的 overlay（tasks 完整面板 / music 完整控件 / settings 面板）**同时只允许打开一个**。点击另一个会先关上当前那个。这是让 Yua 始终是视觉主角的硬规则。

## 6. 风险与权衡

> **决议小结（用户已拍板）**
> - §1 → **保留拖拽 resize**（重做样式让它更克制）
> - §3 → **删除 Type Mode 复选框**，改 chip 入口 + inline 出现 type box（用户原话："感觉就是能 type 的时候再出来 type box"）
> - §7 → **`SelfParticipantChip` 砍掉**，不进 settings、不在 Stage 2 出现
> - §9 → **走 string map**（新元素 + 现有 UI label 一起迁）
> - Stage 2 顺序 → **同意从根节点整理 + Theme 骨架开始**

**1. ~~取消~~ → 保留 tasks 拖拽 resize handle**
现在 `TodoResizeHandle` 允许用户自定义 tasks 面板尺寸并持久化到 save。**保留这个能力**。Stage 2 实现要点：(a) 拖把手位置随 panel 展开态而出现，折叠 tab 态隐藏；(b) 拖把手样式重做成 8×40 的暖灰圆角小握把（垂直三条短线 grip 视觉），不再像现在那样一个醒目按钮；(c) 持久化逻辑不动（继续写 `user://player_profile.json` 里的 `todo_panel_offsets`）；(d) 拖拽范围依然受 `TODO_PANEL_MIN/MAX_WIDTH/HEIGHT` 约束。

**2. 单一工具展开规则（有争议）**
提案规定 tasks / music / settings 同时只能展开一个。好处：Yua 永远是视觉主角；坏处：边写任务边调音乐需要展开两次。取舍接受，因为这是 focus companion 游戏，不是 productivity dashboard——分心成本应该有一点点摩擦。

**3. 移除 `AIModeToggle` CheckButton（已确认）**
"Type Mode" 复选框删除，改 chip `[ 自己打字… ]` + 底部 `💬` icon 两个入口。**Type box 只在被触发时才 inline 出现**（用户原话："感觉就是能 type 的时候再出来 type box"），不再作为常驻控件。实现要点：(a) `PlayerInput` + `SendButton` 默认 `visible=false`；(b) 点 `[ 自己打字… ]` 或 `💬` 时，inline 替换当前 chips 区域，输入框获得焦点；(c) 输入框出现时旁边有一个小的关闭 `✕` 让玩家退回到选项 chips；(d) 提交后输入框自动收回。新玩家发现性风险的缓解：第一次显示对话选项时给 `[ 自己打字… ]` chip 一次轻 pulse 高亮。

**4. Call control bar 6 个 icon 的功能现在大多是装饰**
`🎤 静音` `🖥 共享屏幕` `📞 结束通话` 在 Stage 2 接的时候大概率只是 stub（screenshare 可能完全没逻辑）。风险：玩家点了发现没反应会觉得 bug。取舍：(a) screenshare 直接显示 `还在路上 · 即将上线` ghost toast；(b) end call 接 `quit confirm modal`（真的退出游戏）；(c) mute 实接 BGM/voice mute；(d) volume 实接 master volume；(e) chat 实接对话区折叠；(f) settings 实接现有设置面板。这样 6 个里 5 个真的有反应，只有 screenshare 是占位——可接受。

**5. 自然语言中的"通话感"不要走太远**
做太多 Zoom 化的元素（参与者列表、聊天滚动、emoji reactions、call timer 分段记录）会变成在做 Discord 而不是游戏。守住：**只做点缀，不做模仿**。如果之后某个具体 icon 真的需要复杂功能（例如 chat scrollback），单独开 session 讨论是否值得。

**6. LXGW WenKai 10MB 字体打包成本**
