# 当前 Demo 脚本流程图

> 这份图只描述**目前已经实现进游戏的脚本结构**，不是最终叙事设计稿。  
> 现在的实现更像“专注循环主干 + 少量关系推进节点 + 若干侧支”，真正的长线主线还很薄。

## 1. 总览：当前脚本到底由什么组成

```mermaid
flowchart TD
    A["进入游戏"] --> B["入口分流"]
    B --> C["主线 / 专注循环"]
    C --> D["专注完成后的关系推进"]
    C --> E["专注中戳 Yua 的反应"]
    C --> F["玩家自由输入 / Type Mode"]
    C --> G["中断、离开、回流"]
```

## 2. 入口分流

```mermaid
flowchart TD
    A["启动游戏"] --> B{"demo_script_version_seen < 4 ?"}
    B -- "是" --> C["强制进入 first_launch_01<br/>并重置已完成 session 数、当前任务"]
    B -- "否" --> D{"是否从未看过开场?"}
    D -- "是" --> C
    D -- "否" --> E{"距上次离开 <= 30 分钟?"}
    E -- "是" --> F["return_open_short<br/>短离开后回来"]
    E -- "否" --> G{"当前时段?"}
    G --> H["morning greeting 01-08"]
    G --> I["noon greeting 01-08"]
    G --> J["evening greeting 01-08"]
    G --> K["night greeting 01-08"]
    H --> L["TASK_INPUT_001 或 quiet_end_01"]
    I --> L
    J --> L
    K --> L
    F --> M["继续刚才的 -> ACTION_QUICK_FOCUS"]
    F --> N["换任务 -> TASK_INPUT_001"]
    F --> O["重新体验开场 -> first_launch_01"]
```

### 当前入口触发条件

- **版本升级首次启动**：`demo_script_version_seen < DEMO_SCRIPT_VERSION`，当前版本是 `4`。
- **真正第一次启动**：`has_seen_intro == false`。
- **短时回流**：距离上次离开不超过 `30 分钟` 时，进入 `return_open_short`。
- **普通回流**：否则按现实时间进入早 / 中 / 晚 / 夜问候节点。

## 3. 主线：目前真正存在的“剧情推进”

```mermaid
flowchart TD
    A["first_launch_01"] --> B["first_launch_02"]
    B --> C["first_launch_03<br/>工作 / 假装工作 / 还没想好"]
    C --> D["三种开场回应"]
    D --> E["TASK_INPUT_001<br/>第一次输入任务"]
    E --> F["玩家输入任务文字"]
    F --> G["TASK_INPUT_002<br/>选时长"]
    G --> H["3 秒 / 15 分钟 / 25 分钟 / 45 分钟"]
    H --> I["FOCUS_READY_001"]
    I --> J["ACTION_START_FOCUS"]
    J --> K["FOCUS_START_001"]
    K --> L{"专注结束方式"}
    L -- "自然完成" --> M{"completed_focus_sessions"}
    L -- "手动停止" --> N["ABORT_001"]

    M -- "第 1 次" --> O["FOCUS_DONE_001"]
    M -- "第 2 次" --> P["REL_001"]
    M -- "第 3 次及以后" --> Q["FOCUS_DONE_REPEAT"]

    O --> R["BREAK_CHAT_001<br/>第一次复盘"]
    P --> S["REL_OK / REL_DRAG / REL_TOGETHER"]
    S --> T["BREAK_CHAT_REL"]
    Q --> U["BREAK_CHAT_REPEAT"]

    R --> V["ACTION_QUICK_FOCUS 或 EXIT_DONE"]
    T --> V
    U --> W["ACTION_QUICK_FOCUS / AI_MODE_BREAK_CHAT / EXIT_DONE"]
    V --> X["后续直接继续专注"]
```

### 目前的主线层级

1. **第一次完成 session**：她第一次承认“你来了之后，我也真的做了点事”。
2. **第二次完成 session**：进入 `REL_001`，第一次明确把“有人一起做事会不一样”说出来。
3. **第三次以后**：目前没有新剧情，只进入重复完成节点 `FOCUS_DONE_REPEAT`。

### 这意味着什么

当前 demo **还没有真正展开主线**。  
它只有：

- 开场建立关系
- 第一次共同行动后的轻微靠近
- 第二次共同行动后的第一次关系确认

从第 3 次 session 开始，已经回到“循环玩法文案”，还没进入更深的故事推进。

## 4. “之后不用再报任务”的当前实现

```mermaid
flowchart TD
    A["第一次输入任务后"] --> B["current_focus_task 已保存"]
    B --> C{"玩家从哪里继续?"}
    C --> D["FOCUS_DONE_001: 再来一轮"]
    C --> E["BREAK_CHAT_*: 接着来 / 再来一次"]
    C --> F["return_open_short: 继续刚才的"]
    C --> G["return_open_01: 继续上次的任务"]
    D --> H["ACTION_QUICK_FOCUS"]
    E --> H
    F --> H
    G --> H
    H --> I{"是否已有 current_focus_task?"}
    I -- "有" --> J["不再重新问任务<br/>直接继续专注"]
    I -- "没有" --> K["填入默认文案: 继续刚才那件事"]
    J --> L["ACTION_START_FOCUS"]
    K --> L
```

### 这里的设计意图

- **第一次**需要输入任务，因为游戏要知道你这轮大概在做什么。
- **之后**不该每轮都像老师点名，所以通过 `ACTION_QUICK_FOCUS` 直接回到专注。
- 目前这个规则已经接上了，但还没有做得很“生活化”：  
  她现在只是说一句“那就不重新报任务了，我们继续做自己的事”，以后还可以更自然。

## 5. 专注中戳 Yua 的反应线

```mermaid
flowchart TD
    A["专注计时中"] --> B["玩家点击 Yua"]
    B --> C{"focus_click_count <= 3 ?"}
    C -- "是" --> D["随机抽一条专注中短句<br/>FOCUS_CLICK_LINES 池"]
    C -- "否" --> E["只显示：……"]
    D --> F["继续专注"]
    E --> F
```

### 当前触发规则

- 只在 `focus_running == true` 时生效。
- 每轮专注中：
  - 前 `3` 次点击：从 `FOCUS_CLICK_LINES` 里随机抽一句。
  - 第 `4` 次起：她不再聊天，只回 `……`。

### 这条线的功能

- 它不是主线。
- 它是“陪伴感”和“轻微克制”的反馈层。
- 它也在系统层面防止玩家把专注时段变成聊天时段。

## 6. 玩家输入 / Type Mode 相关分支

```mermaid
flowchart TD
    A["玩家输入文字"] --> B{"Type Mode 是否开启?"}
    B -- "否" --> C{"当前节点是否为 TASK_INPUT_001?"}
    C -- "是" --> D["把输入当成任务文本<br/>_capture_focus_task()"]
    C -- "否" --> E["提示：当前是按钮回复模式"]

    B -- "是" --> F{"当前 AI Mode"}
    F --> G["AI_MODE_TASK_CLARIFY<br/>专注前缩小任务"]
    F --> H["AI_MODE_POST_SESSION<br/>完成后简短复盘"]
    F --> I["AI_MODE_BREAK_CHAT<br/>休息时有限闲聊"]
    F --> J["其他保留模式<br/>check-in / memory 等"]
    G --> K["AI 回一句，再回按钮流"]
    H --> K
    I --> K
    J --> K
```

### 当前真正接入到脚本里的 Type Mode 入口

- `TASK_INPUT_001`
  - 按钮：`帮我把任务缩小一点`
  - 进入 `AI_MODE_TASK_CLARIFY`
- `BREAK_CHAT_001`
  - 按钮：`我想自己说`
  - 进入 `AI_MODE_POST_SESSION`
- `BREAK_CHAT_REL`
  - 按钮：`我想自己说几句`
  - 进入 `AI_MODE_POST_SESSION`
- `FOCUS_DONE_REPEAT`
  - 按钮：`自己说几句`
  - 进入 `AI_MODE_POST_SESSION`
- `BREAK_CHAT_REPEAT`
  - 按钮：`自己说几句`
  - 进入 `AI_MODE_BREAK_CHAT`
- `ABORT_REST`
  - 按钮：`聊一会儿`
  - 进入 `AI_MODE_BREAK_CHAT`

### 当前没真正成为剧情推进器的 AI

- AI 现在只是**侧枝**：
  - 任务澄清
  - 完成后反思
  - 休息时有限聊天
- 它**不会**：
  - 解锁主线
  - 替代剧情节点
  - 在专注前无限闲聊

这点目前是符合项目方向的。

## 7. 中断 / 离开 / 回流线

```mermaid
flowchart TD
    A["专注中"] --> B["玩家点击停止"]
    B --> C["ABORT_001"]
    C --> D["想重新来一轮 -> TASK_INPUT_001"]
    C --> E["先离开 -> EXIT_ABORT"]
    C --> F["先休息一下 -> ABORT_REST"]
    F --> G["聊一会儿 -> AI_MODE_BREAK_CHAT"]
    F --> H["重新开始 -> TASK_INPUT_001"]
    F --> I["先离开 -> EXIT_ABORT"]

    J["普通结束"] --> K{"是否深夜?"}
    K -- "是" --> L["EXIT_NIGHT"]
    K -- "否" --> M["EXIT_DONE / EXIT_001"]
```

### 当前触发规则

- **中断专注**：只有计时器正在跑时按停止，才进入 `ABORT_001`。
- **夜间离开**：离开逻辑里会额外判断当前是否夜间，夜里会优先走 `EXIT_NIGHT`。
- **回流**：见上面的入口分流图。

## 8. 用一句话总结当前结构

```mermaid
flowchart LR
    A["第一次来"] --> B["报一个任务"]
    B --> C["一起专注"]
    C --> D["第 1 次完成：她承认你的存在让她也动起来了"]
    D --> E["第 2 次完成：她承认一起做事和一个人不一样"]
    E --> F["第 3 次以后：进入重复循环，主线暂时停止增长"]
```

## 9. 当前结构最明显的问题

1. **主线太薄**  
   第 2 次完成之后，故事就没有继续长了。

2. **开场分支有，但没有带来后续性格化回响**  
   “工作 / 假装工作 / 还没想好”现在只影响当下几句，不会在后面形成记忆感。

3. **Type Mode 已经有位置，但还没有和角色关系形成很聪明的互文**  
   现在它只是功能性插口，还没有成为“她为什么会记住你某些事”的叙事工具。

4. **Poke 线很完整，但它目前只是气氛层**  
   它还没和关系阶段、任务类型、专注次数发生联动。

5. **重复循环比剧情更成熟**  
   从系统角度已经很像一个 focus companion；从“想让玩家继续回来是为了见她”的角度，钩子还不够强。

## 16. 如果按“玩家实际体验顺序”读，当前脚本是这样

1. 她先注意到有人进来了。
2. 她笨拙地自我介绍，说自己也是来找人一起不拖延的。
3. 她问你今天是真工作，还是假装工作。
4. 不管怎么选，最后都会引到第一次任务输入。
5. 你输入任务，选时长，然后一起开始。
6. 第一轮结束后，她第一次承认：你来了，让她也真的做了点事。
7. 如果你聊，她会问你推进得怎样。
8. 第二轮结束后，她第一次把“有人陪会好一点”这件事说出口。
9. 第三轮以后，关系没有继续明显推进，进入稳定陪伴循环。
10. 专注中你可以戳她，但她会克制地把你推回任务上。
11. 你可以在少数节点切到 Type Mode，自由说一句，但 AI 不接管主线。

## 17. 现在这份脚本读下来，最值得注意的节奏事实

1. **最有“角色关系开始发生”的句子，其实集中在 3 处**
   - “你来了，我就有理由正式开始了。”
   - “你在，我就不好意思一直发呆。”
   - “有人在，和一个人坐着，感觉不一样的。”

2. **第一次 completion 的情绪回报比开场更有魅力**
   - 开场还在“产品 onboarding”附近。
   - 第一次完成后，她才开始比较像一个值得继续见的人。

3. **第二次 completion 已经把关系说得有点太直了**
   - `REL_TOGETHER` 很甜，但如果这是第二轮就说到这里，后面会比较难继续升级。
   - 也就是说：**它有效，但有点早花掉了一张好牌。**

4. **第三次以后断崖式回到功能循环**
   - 这就是现在你会觉得“怎么没有推进”的根本原因。

5. **当前最成熟的其实是她在专注中被点时的声音**
   - 那 23 条 poke line 反而最接近一个“有点可爱、有点熟、有点会逗你”的 Yua。
   - 如果要重写整体 tone，那一组可以当作比较好的内部参照物。
